# 아빠, Go! — 캐릭터를 "느낌있게" 만드는 방법 (설계)

> 상태: **승인 대기 (pending approval)** · 모드: /plan Direct · 작성일: 2026-06-24
> 범위: 자료/에셋 **수집 + 설치(연동) 계획**까지만. **소스 코드 편집 금지**(다른 세션 동시작업 충돌 방지).
> 모든 산출물은 `assets/external_v3/` 와 본 계획서에만 기록.

## 0. 질문에 대한 답 — "어떤 방법이 있을까?"
5~9세 정착 게임에서 캐릭터의 "느낌(따뜻함·친근감)"은 **걷는 스프라이트의 디테일이 아니라
대화할 때 보이는 얼굴(포트레이트) + 표정 + 작은 움직임**에서 거의 다 나온다.
현재 데모는 대화창에서 **이모지를 캔버스에 그리는 수준**(`ui.js:39` `drawAvatar`)이라 감정이 없다.
핵심 레버 4개: ① **얼굴 포트레이트**, ② **표정 변화**, ③ **미세 애니메이션/감정 말풍선**, ④ **일관된 캐스트(한 가족의 그림체)**.

## 1. 캐스트(등장 캐릭터)와 "느낌" 목표
| 캐릭터 | 역할 | 느낌 키워드 | 실루엣 단서 |
|------|------|------|------|
| 아빠 | 동행자·기준점 | 든든함, 다정 | 키 큼, 모자/백팩 |
| 아이(플레이어) | 나 | 호기심, 씩씩 | 작은 키, 노란 옷 |
| 새 친구 | 또래 | 명랑 | 비슷한 키, 다른 색 |
| 도서관 선생님 | 안내 | 차분, 따뜻 | 안경, 책 |
| 시장 아주머니 | 생활 | 푸근 | 앞치마 |
| 버스 기사님 | 이동 | 친절 | 제복/모자 |
| 약사님 | 안심 | 부드러움 | 흰 가운 |
| 산책 할아버지 | 동네 어른 | 온화 | 지팡이, 굽은 등 |
| 갈매기 | 속초 상징(유일 판타지) | 귀여움 | 새, 큰 눈 |

## 2. 방법 옵션 (장단점)

### Option A — 무료 CC0/모듈러 스프라이트 + 무료 포트레이트 (비용 0)
- 스프라이트: **Kenney Modular Characters**(CC0, 425개) 또는 보유 LPC를 **팔레트 스왑**해 캐스트 통일.
- 포트레이트: 무료 CC0 픽셀 포트레이트.
- 장점: 무료, 라이선스 안전, 즉시.
- 단점: **검증 결과 무료 픽셀 포트레이트는 판타지/서양 톤**이 많아 "동네 사람·그림책" 느낌과 안 맞음
  (실제로 OGA "39 Portraits"를 받아 확인 → 빨강머리 전사·초록 고블린 → **부적합 판정·삭제**).

### Option B — AI 그림책풍 포트레이트 캐스트 (느낌 최상, 권장)
- 프로젝트에 이미 있는 `generate-assets.mjs`(OpenAI 이미지) 활용 — **고정 스타일 프롬프트 + 시드**로
  9명 캐스트의 **따뜻한 그림책풍 얼굴 컷**을 일관되게 생성. 표정 2~3종(웃음/놀람/말하는 중)도 같이.
- 장점: 5~9세에 가장 따뜻함, 캐스트 톤 100% 통일, 한국 동네 감성 반영 가능.
- 단점: API 키 필요(빌드타임), 실사진 아님, 컷별 일관성은 프롬프트/시드 관리 필요.

### Option C — 모듈러 팔레트 스왑(스프라이트만 통일, 코드 작업 큼)
- LPC/Kenney 모듈러를 코드로 합성해 캐스트 생성. 포트레이트는 별도.
- 장점: 무료, 무한 변형. 단점: 합성 코드 부담, 얼굴 감정은 여전히 약함.

### Option D — 하이브리드 (권장 최종)
- **이동 = 코히런트 스프라이트**(Kenney Modular CC0 또는 LPC 스왑) +
  **대화 = AI 그림책 포트레이트**(Option B) + **미세 애니 + 감정 말풍선 + 표정 동기화**.
- 스프라이트는 "움직임", 포트레이트는 "감정"을 담당 → 비용 최소로 느낌 최대.

## 3. "느낌"을 만드는 기법 (옵션과 무관하게 공통 적용)
1. **대화 포트레이트 컷**: 대화창 좌측에 큰 얼굴(현재 이모지 → 실제 얼굴 이미지). `ui.js` 대화 흐름에 슬롯만 있으면 됨(설치 시).
2. **표정 동기화**: 대사 줄마다 표정 키(`smile`/`surprised`/`talk`) 지정 → 포트레이트 교체.
3. **미세 애니메이션**: idle 위아래 bob(이미 NPC에 `tweens` 패턴 있음 `game.js:199`), 눈 깜빡임, 보상 시 깡총 점프.
4. **감정 말풍선**: 머리 위 `!` `?` `♥` `♪` 아이콘 — 근접/대화/보상 순간에.
5. **거리별 실루엣 차별화**: 키·소품(모자·지팡이·앞치마)로 멀리서도 누가 누군지.
6. **목소리 매칭**: 이미 `npc.pitch/rate`로 음색 차별화 중(`data/places.js`) → 포트레이트와 톤 일치.

## 4. 수집 현황 / 추가 수집 계획
- ✅ 보유: LPC 워크사이클(`assets/external_v2/characters/*`) + Kenney Urban 주민 스프라이트(`external_v3` 시트 우측).
- ⬜ 추가(설치 단계에서 결정 후):
  - Option A 선택 시: **Kenney Modular Characters** zip(CC0) — https://kenney.nl/assets/modular-characters
  - Option B 선택 시: `generate-assets.mjs`에 캐스트 프롬프트표 추가(별도 세션/승인 후, 본 세션은 미실행).
- ❌ 보류: OGA "39 Portraits"(OGA-BY, 판타지 톤) — **부적합 확인·미사용**.

## 5. 설치(연동) 계획 — 코드는 다른 세션이 작업
> 본 세션은 **파일을 만들지 않고**, 어디에 무엇을 넣을지 "설치 지점"만 명세한다.
1. **포트레이트 폴더 규약**: `assets/portraits/<charId>_<expr>.png` (예: `dad_smile.png`, `gull_happy.png`).
2. **데이터 연결 지점**: `data/places.js`의 각 `npc`에 `portrait` 베이스명 + 대사별 `expr` 추가(설치 세션이 수행).
3. **렌더 지점**: `ui.js:39 drawAvatar()`를 이모지→`portrait` 이미지 로드로 교체(없으면 이모지 폴백 유지).
4. **말풍선/애니**: `game.js` NPC 생성부에 emote 텍스트 + tween(기존 패턴 재사용).
5. **폴백 원칙 유지**: 이미지 없으면 현재 이모지로 자동 대체 → 빈 채로도 동작.

## 6. 인수 기준 (테스트 가능)
- [ ] 대화창에 **이모지가 아니라 캐릭터 얼굴 포트레이트**가 표시된다(없으면 이모지 폴백).
- [ ] 같은 대화에서 **대사에 따라 표정이 최소 2종 바뀐다**.
- [ ] 근접/보상 순간 **감정 말풍선(!·♥ 등)**이 뜬다.
- [ ] 9명 캐스트가 **한 그림체(톤·외곽선·채도)**로 통일돼 보인다.
- [ ] 모든 캐릭터 에셋의 라이선스가 `CREDITS`에 기록된다(CC0 우선; AI생성은 생성 고지).
- [ ] 실제 아동 사진/식별 가능한 실제 인물 미사용(개인정보).

## 7. 리스크 / 완화
| 리스크 | 완화 |
|------|------|
| 무료 포트레이트 톤 불일치(검증됨) | Option B(AI 그림책) 또는 한 팩으로 통일, 판타지 팩 배제 |
| AI 컷 일관성 흔들림 | 고정 스타일 프롬프트+시드+동일 구도, 9명 한 배치 생성 |
| LPC(외곽선 굵음) vs Kenney(부드러움) 혼용 | 스프라이트는 한 계열로만, 포트레이트는 별도 계열 OK |
| 본 세션과 코드 세션 충돌 | 본 세션은 `assets/`·계획서만 기록, 소스 미편집 |

## 8. 권장
**Option D(하이브리드)** — 스프라이트는 보유 LPC/Kenney로 무료 통일, **대화 포트레이트만 AI 그림책풍으로
9명+표정 생성**. 느낌의 90%는 포트레이트+표정+말풍선에서 나오므로 비용 대비 효과 최대.
다음 단계로 Option A/B/D 중 선택을 확정하면, 그에 맞는 에셋만 추가 수집한다(코드 연동은 담당 세션).
