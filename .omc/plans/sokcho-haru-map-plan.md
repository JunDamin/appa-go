## 0. 핵심 설계 문장
> 아빠, Go!의 속초 맵은 관광 명소 지도가 아니라, **아이의 하루가 흘러가는 생활 지도**다.
> 집에서 출발해 학교 가는 길을 익히고, 방과후 도서관·놀이터를 발견하고, 저녁엔 시장·호수
> 산책길을 경험한다. 바다·문화유산은 특별한 주말 탐험지다.
> **탐험은 호기심을 만들고, 일상은 친근감을 만든다.**
> 한 문장: *새 동네를 모험처럼 발견하되, 결국 "내가 살 하루"로 느끼게 한다.*

## 1. 요구사항 요약

사용자의 가장 중요한 구분: **"기능이 도는 것 ≠ 맵이 잘 구성된 것."**
현재 데모는 절차 생성 단색 타일 4종 + 사각형 그래픽 건물 + 이모지 NPC라 **동작은 하지만
시각적으로 빈약**하다. 이번 작업은 세 가지를 **별도 산출물**로 설계한다.

- **A. 에셋(Assets)**: 실제 CC0 타일/건물/소품/캐릭터/사진/일상카드 — 무엇을, 어디서, 어떻게.
- **B. 맵 구성(Map)**: 절차 생성 → **손배치(전이타일+장식레이어)** 기반 "잘 구성된" 속초 하루지도.
- **C. 미션/시나리오**: 기대 미션 **전부** 노출 + 친근감(정착감)이 도는 하루 루트 시나리오.

비기능 목표: 5~9세 + 보호자, 모바일/태블릿, 음성 읽어주기, 개인정보 보호(실제 학교/아이 사진 금지).

## 3. A. 에셋 설계 (Assets)

### 3.1 확보 완료 (이미 assets/에 배치)
- ✅ `assets/external_v3/` — **Kenney RPG Urban Pack**(CC0, 486 타일 16px): 건물 지붕/벽/문/창, 도로, **횡단보도**, 가로등, 벤치, 울타리, 표지판, 나무, **자동차**, 4방향 주민 캐릭터. → 맵에 빠져 있던 **실제 건물/소품**을 채움.

### 3.2 에셋 팔레트 — 구역별 사용 매핑

## 4. B. 맵 구성 설계 (Map) — "잘 구성된" 핵심

**원칙: 순수 절차 생성 → 데이터 주도 손배치.** 절차 생성은 단조롭고 전이가 없어 "관광 게임"처럼
보인다. 잘 구성된 맵은 ① 전이 타일(잔디↔물↔모래↔도로 경계), ② 실제 건물 스프라이트, ③ 장식
레이어(가로등/벤치/나무/횡단보도/표지판), ④ 의미 있는 동선이 필요하다.

### 4.1 레이어 구조 (3레이어)
1. **ground**: 잔디/물/모래/포장 + **전이 타일**(external_v2 LPC 오토타일 모서리 사용).
2. **deco**: 건물 스프라이트(external_v3), 가로등·벤치·울타리·횡단보도·표지판·나무·자동차.
3. **entities**: 플레이어, NPC, 포탈존(깊이정렬 y기준 — 기존 `game.js:214` 유지).

### 4.2 데이터 주도 맵 정의 — `data/tilemap-sokcho.js`(신규)
절차 `worldDef()`를 **하이브리드**로 교체: 베이스 지형은 절차(바다/호수/산), 그러나
**도로·건물·장식은 명시적 좌표 배열**로 손배치. 예시 스키마:
```js
world = {
  cols: 48, rows: 40,
  regions: { home:{x,y,w,h}, school:{...}, afterschool:{...}, life:{...}, walk:{...}, special:{...} },
  terrain: [...],                 // 절차 베이스(바다 동, 청초호 중앙, 영랑호 북동, 설악 서)
  roads: [ {from:'home', to:'school'}, ... ],   // 하루 루트 동선을 따라 도로
  deco: [ {tile:'lamp', x,y}, {tile:'crosswalk', x,y}, {tile:'bench', x,y}, ... ],
  buildings: [ {placeId:'school', sprite:'urban:school', x,y}, ... ],
}
```

### 4.3 속초 지리 반영 (실제 구조 유지)
서=설악산(숲) · 동=동해/해수욕장 · 중앙=청초호(석호) · 북동=영랑호 · 북중앙=시장·동명항 ·
남(조양동 신시가지)=우리집·학교·도서관·마트·놀이터. (현 `DESIGN.md` 지도 구조 계승)

### 4.4 하루 동선이 보이는 맵
도로/포장 동선이 **집→학교→방과후→생활→산책→집→(주말)바다** 흐름을 시각적으로 그림.
안개(fog) 오버레이로 미발견 구역을 가리고, 방문 시 페이드로 걷힌다(대항해시대식 발견감).

### 4.5 두 모드 유지 (기존 계승)
- 월드맵(동네 탐험): 카메라 추적 걷기.
- 장소 상세맵(항구 모드): 건물 진입 → NPC 대화 → 사진+찾기미션 → 퀘스트 → 조각+일상카드 → 복귀.
  상세맵도 external_v3 실내/파사드 타일로 테마별로 꾸민다.

## 5. C. 미션 / 시나리오 설계 — 기대 미션 "전부" + 친근감

### 5.1 메인 목표 변경
- 기존: "속초 보물지도 완성" → 신규: **"나의 속초 하루지도 만들기"**.
- 보물지도(탐험감)는 유지하되 **결과 화면은 일상 지도**로 보여준다.

### 5.2 하루 루트 (반자유 — 사용자 확정)
```
집 → 학교 가는 길 → 학교 → [도서관 | 놀이터 선택] → [시장 | 청초호 선택] → 집 → (주말 바다 해금)
```
큰 흐름은 하루 순서로 안내, 각 단계에서 **선택지**로 자유 탐험. 숨은 탐험지(조양동 유적·아바이마을·영랑호·영금정)는 보너스로 항상 개방.

기존 6장소(school/daycare/library/mart/playground/market)는 일상 표현·일상카드만 보강.

### 5.4 미션 1 상세 예시 (나머지는 동일 패턴으로 `places.js` 확장)
- 일상 표현: "오늘은 새 학교 가는 길을 걸어봐요."
- 아빠 대사: "학교까지 길을 같이 걸어보자. 어떤 표시를 기억하면 돌아올 때 안심될까?"
- 사진 보기: 횡단보도·큰 길·학교 근처 길·버스정류장
- 찾기 미션: 횡단보도 찾기 / 멈춰야 하는 곳 / 아빠와 만날 장소
- 보상: 길 기억 카드 🗺️ + 학교 조각 진행

### 5.5 NPC = 동네 사람 (판타지 최소화)
아빠(동행자·기준점), 새 친구, 도서관 선생님, 시장 아주머니, 버스 기사님, 약사님, 산책 할아버지 +
**갈매기 1종만 판타지 상징**. 나머지는 친절한 동네 어른 → 일상감.

### 5.6 결과 화면 (친근감 마무리)
```

저녁에는 아빠와 시장에서 간식을 사요.  주말에는 청초호를 걷거나 바다를 봐요.
```

+ 모은 일상카드 8장 + 보물조각을 "하루지도" 위에 배치해 보여준다.

## 6. 구현 단계 (실행 승인 시)

1. **데이터 확장** `data/places.js`: 신규 4장소(버스정류장·약국·청초호산책·해수욕장) 추가, 각 장소에 `dailyCard`·`timeSlot`·`settleProblem` 필드 추가, 대사를 일상 표현으로 보강.
2. **에셋 인덱스 맵** `data/tiles-sokcho.js`: external_v2+v3 시트의 용도별 타일 인덱스 표 작성(grass/water/sand/road/crosswalk/lamp/bench/roof…).
3. **맵 데이터** `data/tilemap-sokcho.js`: §4.2 스키마로 구역·도로·deco·buildings 손배치.
4. **렌더 교체** `game.js`: `addBuilding()` 절차 그래픽 → external_v3 건물 스프라이트, deco 레이어 추가, 안개(fog) 오버레이, NPC를 주민 스프라이트로.
5. **진행 시스템** `ui.js` + 신규 `data/day-route.js`: 하루 루트 게이팅(반자유), 8칸 미션 보드 HUD, 일상카드 수집/표시, 결과=하루지도.
6. **사진/오디오**: §3.4 일상사진 교체(있으면), BGM/SFX 유지.
7. **추가 에셋 소싱**: §3.3 놀이터기구·물결 타일.

## 7. 인수 기준 (Acceptance Criteria — 테스트 가능)

- [ ] 월드맵이 절차 단색 4타일이 아니라 **전이 타일 + 실제 건물 스프라이트 + 장식 레이어**로 렌더된다(육안: 가로등·벤치·횡단보도·자동차·나무가 보임).
- [ ] **8개 미션이 모두** 진행 보드 HUD에 노출되고, 각 미션 완료 시 칸이 채워진다.
- [ ] 하루 루트가 집→학교→방과후(선택)→생활(선택)→집→주말바다 순으로 게이팅되며, 각 단계에서 선택지가 동작한다.
- [ ] 각 장소에서 **보물조각 + 일상카드 둘 다** 획득되고 HUD/결과에 반영된다.
- [ ] 결과 화면이 "나의 속초 하루지도" 문구 + 카드 8장으로 표시된다.
- [ ] NPC가 이모지가 아니라 주민 스프라이트(또는 일관된 캐릭터 아트)로 보인다.
- [ ] 모든 에셋이 CC0/공공누리/CC 출처와 함께 `CREDITS`에 기록된다.
- [ ] 실제 학교/아이 사진이 사용되지 않는다(개인정보).
