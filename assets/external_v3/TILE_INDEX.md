# TILE_INDEX — Kenney "RPG Urban" 용도별 타일 인덱스

**시트**: `urban_tilemap_packed.png` — 432×288px, **27열 × 18행 = 486 타일, 16×16px, 무간격**.
`index = row*27 + col` (0..485, 행우선). 개별 `urban_tiles/tile_NNNN.png` 파일번호 = 그 index.

Phaser 로드:
```js
this.load.spritesheet('urban', 'assets/external_v3/urban_tilemap_packed.png',
  { frameWidth:16, frameHeight:16 });
```
바로 붙여쓰는 JS 객체는 `data/urban-tiles.js` (`globalThis.URBAN_TILES`) 참고.

검증 방법: 486개 개별 PNG를 순수 node(zlib) 디코더로 평균색 분석 + 시트를 10~24배 확대해 라벨 붙여 육안 대조.

## CHARACTERS / 주민 NPC  ★ 애니메이션 판정

**위치**: 시트 **우측 4열(col 23~26), 18행 전체 = 72타일 = 6명**.
(미리보기 montage는 간격을 넓혀 더 많아 보이지만, packed 시트에선 정확히 우측 4열.)

**프레임 레이아웃 (캐릭터 1명 = 3행 × 4열 = 12프레임)**
- **열(방향)**: col23=왼쪽 · col24=아래(정면) · col25=위(뒷모습) · col26=오른쪽 → **4방향**
- **행(워크)**: row0=정지/서기 · row1=걷기A · row2=걷기B → **방향당 3프레임 워크**

### ▶ 판정: **단일정지 아님 — 진짜 4방향 워크 애니메이션 가능** (RPG-Maker식 3프레임/방향)
- 같은 캐릭터의 3행을 비교하면 다리/팔이 명확히 (모음→한쪽내딛음→반대내딛음)으로 바뀜 → 걷기 사이클.
- LPC처럼 한 줄에 8프레임은 아니지만, **걷기 3프레임 × 4방향**으로 충분히 walk-cycle 구동.
- (bob/hop tween 폴백은 불필요. 3프레임 ping-pong: 정지→A→정지→B 권장.)

### 프레임 헬퍼

예) childGreen 정면(down=base+1=24)의 워크 3프레임 = **24, 51, 78**.
